# Paper Quality And Submission Workflow

Use this file when the user wants to produce, audit, or submit a high-quality paper.

Do not print this whole workflow by default. Use it internally unless the user asks for full paper mode, submission readiness, or systematic audit.

## Publication-Oriented Quality Checks

High-quality papers need:

- a narrow and defensible knowledge gap;
- one central claim, not a list of analyses;
- a clear evidence chain from user data to claim;
- figures that each perform a specific argumentative job;
- methods that a reviewer can reproduce and defend;
- cautious interpretation proportional to task, population, and method;
- explicit limitations;
- journal fit and submission strategy.

## Paper Spine

Use this compact structure:

```text
Central problem:
Knowledge gap:
Main claim:
Approach:
Key evidence:
Boundary:
Contribution:
Target journal or audience:
Main reviewer risk:
```

For short answers, compress this into one paragraph rather than printing the template.

## Claim-Evidence Register

Use when drafting discussion, introduction, abstract, or response letters.

```text
Claim:
User data anchor:
Citation anchor:
Support level:
Boundary:
Reviewer risk:
Action:
```

Support levels:

- direct;
- partial;
- background;
- contradictory;
- unsupported.

Unsupported claims must be deleted, weakened, or converted to future-work language.

## Figure-To-Argument Check

For every major figure:

```text
Figure:
What it literally shows:
Which claim it supports:
What the reader should notice:
What the caption must state:
Potential reviewer concern:
Needed companion analysis:
```

If a figure does not support a claim, it should be revised, moved to supplement, or removed.

## Reviewer-Risk Simulation

For manuscript-facing claims, ask:

- Is the research question clear and answerable?
- Are groups or continuous scores justified?
- Is the statistical unit correct?
- Are time windows, K values, and contrasts justified independently of the desired result?
- Are multiple comparisons controlled?
- Are trial counts and preprocessing choices transparent?
- Are scalp topographies overinterpreted as source-level mechanisms?
- Are developmental and risk-group claims non-stigmatizing and bounded?
- Does behavior support the cognitive interpretation?

Output only the strongest risks unless the user asks for a full simulation.

## Journal Targeting

When the user asks where to submit, assess:

```text
Scope fit:
Novelty threshold:
Methodological expectations:
Likely reviewer expertise:
Word/figure constraints:
Open science expectations:
Submission risk:
Backup journals:
```

Do not recommend a journal without checking current aims, scope, article types, and recent comparable papers.

## Submission Package

When requested, help prepare:

- title options;
- abstract;
- highlights;
- cover letter;
- graphical abstract idea;
- data availability statement;
- code availability statement;
- ethics statement;
- limitations paragraph;
- reviewer suggestions or exclusions;
- response-to-reviewers structure.

## Default Concise Paper Advice

Unless the user asks for full paper mode, answer in this compact form:

```text
Core publication judgment
Main evidence requirement
Main reviewer risk
Recommended next action
```
